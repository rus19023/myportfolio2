What is ETL?

ETL is an acronym for Extract, Transform, Load. It is the process used with cloud or on-premise data warehouses to: Extract data from one or more sources, Transform it to match the standards needed for the repository, and Load it into said data warehouse repository. A newer take on ETL is ELT, which stands for Extract, Load, Transform. This is a similar process, only the data is loaded first, then transformed. The purpose is to mitigate coding error, migration speed and server scaling issues during the transformation procedure.

Extract

As there may be many differently formatted data sources, extraction can be a difficult task. Data from each source must be profiled to determine if the data content, type and structure is workable with the business' needs. Version control helps monitor for changes in the data between sessions of exportation.

Transform

Extracted data that is considered "dirty", or detracting from the integrity of the data, must now be "cleaned up", removing duplicate, poorly parsed and/or inconsistent data, correcting misspellings. Data changes tracked can be adjusted as needed, then the data is reformatted to make it uniform for reporting and analysis.

Load

Data collected and structured can now be inserted into fact and dimension tables where it is stored in data cubes that can be "sliced and diced" for reports and analysis as needed.

What programming tools does ETL use?

There are many different programming languages, tools and methods used to perform ETL, which may or may not be part of an integrated warehousing system. They include: downloadable software or cloud services for data manipulation, security, policy compliance, version control, data integration, task automation, report generation and the list goes on and on. These tools may be commercially owned or open-source. Programming languages commonly used for ETL tools are Java, Python, R and Scala. (This is not a comprehensive list.)

  • Amazon Redshift: Part of AWS, a column-oriented DBMS, up to 16 PB (petabytes) of data can be stored in a single cluster.
  • Apache Kudu: Part of Apache's Hadoop, a column-oriented data store, optimized for OLAP, open-source.
  • IBM InfoSphere DataStage: Part of IBM Information Platforms Solution, uses graphical notation for data integration, client/server architecture.
  • Microsoft Analysis Services: Part of Microsoft's SQL Server, a tool with two options- multidimensional or tabular, for OLAP and data mining.
  • Oracle Warehouse Builder: Complete service for data warehousing, includes data quality, integration and metadata management.
  • Snowflake Data Cloud.: Fully integrated PaaS, offers "near unlimited scale" in their "data cloud" for large enterprises to "access", "govern" and "action" data.
  • Teradata Vantage: Multi-cloud, hybrid data analysis platform, geared toward scaling for the biggest "big data" enterprises, based on machine.
